  1. HTTP在客户端-服务器计算模型中作为请求-响应协议运行。客户端向服务器提交一个HTTP请求消息。服务器提供HTML文件和其他内容等资源，或者代表客户端执行其他功能，并将一个响应消息返回给客户端。响应包含关于请求完成状态的信息，也可能在其消息正文中包含请求的内容。

    • 1XX (informational) The request was received, continuing process.
    • 2XX (successful) The request was successfully received, understood, and accepted.
    • 3XX (redirection) Further action needs to be taken in order to complete the request.
    • 4XX (client error) The request contains bad syntax or cannot be fulfilled.
    • 5XX (server error) The server failed to fulfill an apparently valid request.

数据库

  1. 关系模型（relational model）。关系模型用表的集合来表示数据和数据间的联系。每个表有多个列，每列有唯一的列名。表也称作关系。关系模型是基于记录的模型的一种。基于记录的模型的名称的由来是数据库是由若干种固定格式的记录构成的。
    每个表包含某种特定类型的记录。每个记录类型定义了固定数目的字段或属性。表的列对应于记录类型的属性。关系数据模型是使用最广泛的数据模型，当今大量的数据库系统都基于这种关系模型。
  2. 数据库系统提供数据定义语言（Data-Definition Language，DDL） 来定义数据库模式，并提供数据操纵语言（Data-Manipulation Language,DML）来表达数据库的查询和更新。
    而实际上，数据定义和数据操纵语言并不是两种互相分离的语言，相反地，它们仅仅是构成了单一的数据库语言（例如SQL 语言）的不同部分。几乎所有的关系数据库系统都使用 SQL语言，我们将在第3~5 章详细介绍 SQL 语言。
  3. 宿主语言（如 C/C++、Java 或 Python）编写，并嵌入访问数据库中数据的 SQL 查询。以这种方式与数据库交互的程序称为应用程序。
  4. 数据库系统被划分为处理整个系统各项职责的模块。其功能组件可分为存储管理器、查询处理器组件和事务管理组件。
  5. 超码（superkey）是一个或多个属性的集合，将这些属性组合在一起可以允许我们在一个关系中唯一地标识出一个元组。

关系代数

  1. select: $\sigma$
    $σ_{dept_name = “Physics” ∧ salary > 90000}(instructor)$
  2. project: $\Pi$
    它返回输入关系，但会去掉某些指定的属性。$Π_{ID, name, salary} (instructor)$ 。投影操作的结果会去除重复的行。
  3. Cartesian-Product: $×$
    会将 “instructor” 关系中的每一个元组与 “teaches” 关系中的每一个元组进行组合，生成新的元组。
    join: $r⋈_θs=σ_θ (r × s)$
  4. Union: $∪$ || Set-Intersection: $∩
    进行并操作的两个关系必须具有相同的 arity（即属性数量相同），并且相应属性的域必须兼容
  5. Set Difference: $-$
    它可以找出在一个关系中存在，但在另一个关系中不存在的元组。进行差操作的两个关系同样要满足属性数量和属性域的兼容性要求。
  6. assignment: $←$
  7. Rename: $ρ$
    $ρ_x(E)$返回以x命名的表达式E的结果。
    $ρ_{x(A1,A2…An）}（E）$返回以x命名的表达式E的结果，并将其属性重命名为A1A2

DDL

  1. create table
  • primary key （Aj, Aj2，，Am）：主码声明表示属性Aj,Ap2，…，Ajm构成关系的主码。主码属性必须是非空且唯一的；也就是说，没有元组会在主码属性上取空值，并且关系中也没有两个元组会在所有主码属性上取值都相同。虽然主码声明是可选的，但为每个关系指定一个主码通常不失为一个好主意。
  • foreign key （Aki, Ak2，…，Ak）references s ：外码声明表示关系中任意元组在属性（AA1，Ak2，…，Ak）上的取值必须对应于关系s中某元组在主码属性上的取值。
  • drop table 命令从数据库中删除关于被去掉关系的所有信息。不仅删除所有元组，还删除r的模式; drop table r保留关系r，但删除r中的所有元组。
  • alter table 命令为已有关系增加属性。关系中的所有元组在新属性上的取值将被赋为 null。alter table 命令的格式为：
    • alter table r add A D;其中，r是现有关系的名称，A是待添加属性的名称，D是待添加属性的类型。
    • alter table r drop A;从关系中去掉属性。r是现有关系的名称，A是关系的一个属性的名称。

statement

Statement主要用于执行那些不包含参数占位符的 SQL 语句

PreparedStatement

PreparedStatement在Statement的基础上增加了设置输入参数的功能，这在处理动态 SQL 时非常实用，不仅能提高代码的安全性，防止 SQL 注入攻击，还能提升性能。

CallableStatement

CallableStatement则是专门用来调用数据库存储过程的，它还能获取存储过程返回的输出参数值。

处理结果

ResultSet用于表示查询结果集，它有不同的类型和并发模式。
TYPE_FORWARD_ONLY 表示结果集只能向前遍历
TYPE_SCROLL_INSENSITIVE允许随意滚动且不受其他事务修改的影响
TYPE_SCROLL_SENSITIVE
CONCUR_READ_ONLY 是只读模式
CONCUR_UPDATABLE支持更新数据
我们可以使用next()、previous()等方法在结果集中移动游标
还能通过findColumn()找到指定列的索引，利用getMetaData()获取结果集的元数据信息。
ResultSet还支持更新、删除和插入行的操作，但这些操作都有特定的步骤和要求。

JDBC template

在执行核心 JDBC 操作时，应用代码只需提供 SQL 语句和提取结果的逻辑，其余复杂操作都由 JdbcTemplate 负责。例如，它能执行 SQL 查询或更新、遍历结果集，还会捕获 JDBC 异常并转化为更具通用性和信息性的异常。
